# `meta/` — working agreements & continuity

This folder is the **durable, in-repo record of how we work on `ccs`** so that
conventions and decisions carry over from session to session (independent of any
assistant's private memory). It is tracked in git and meant to be read at the
start of a working session.

| file | what it holds |
|---|---|
| [`conventions.md`](conventions.md) | data-governance, writing, charts, and precision rules the project follows |
| [`workflow.md`](workflow.md)       | git workflow & how much autonomy the assistant has (branches only; never merge to `main`) |
| [`decisions.md`](decisions.md)     | a dated log of notable decisions and the reasoning behind them |

**If you change how the project works, update the relevant file here in the same
change.** These are not aspirational — they describe the actual, current rules.
